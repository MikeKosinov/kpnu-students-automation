ТЕСТ 1: Негативний сценарій логіну з невірними credentials
===============================================================================

Назва тесту: Login with invalid credentials

Мета: Перевірити, що система правильно обробляє спробу входу з невірними даними

Структура (Gherkin):

GIVEN (Передумови):
- Користувач знаходиться на головній сторінці
- Користувач не авторизований

WHEN (Дії):
1. Користувач натискає на посилання "Signup / Login"
2. Користувач потрапляє на сторінку логіну (URL містить /login)
3. Користувач заповнює поле email невірними даними: "wrong_email@test.com"
4. Користувач заповнює поле password невірними даними: "WrongPassword123"
5. Користувач натискає кнопку "Login"

THEN (Очікувані результати):
1. Користувач залишається на сторінці /login
2. З'являється повідомлення про помилку: "Your email or password is incorrect!"
3. Поля форми залишаються порожніми або очищаються
4. Користувач НЕ авторизований (немає тексту "Logged in as" у header)

Додаткові перевірки:
- Перевірити видимість повідомлення про помилку
- Перевірити, що URL не змінився на домашню сторінку
- Перевірити відсутність імені користувача в header


===============================================================================
ТЕСТ 2: Перевірка форми реєстрації (signup form validation)
===============================================================================

Назва тесту: Signup form validation with empty fields

Мета: Перевірити валідацію обов'язкових полів форми реєстрації

Структура (AAA):

ARRANGE (Підготовка):
- Користувач на головній сторінці
- Форма signup не заповнена

ACT (Виконання):
1. Відкрити головну сторінку
2. Клікнути на "Signup / Login"
3. Перевірити видимість форми "New User Signup!"
4. Спробувати натиснути кнопку Signup БЕЗ заповнення полів
5. Заповнити тільки Name, залишити Email порожнім
6. Спробувати натиснути кнопку Signup

ASSERT (Перевірка):
1. Кнопка signup не активна без заповнення обов'язкових полів
2. З'являються валідаційні повідомлення для порожніх полів
3. Форма не відправляється з порожнім email
4. Перехід на наступну сторінку НЕ відбувається


===============================================================================
ТЕСТ 3: Повний flow реєстрації з logout (позитивний + cleanup)
===============================================================================

Назва тесту: Complete registration flow with logout

Мета: Перевірити повний цикл реєстрації користувача та виходу з системи

Структура (Gherkin Extended):

GIVEN (Передумови):
- Користувач на головній сторінці
- Email для реєстрації унікальний (згенерований)

WHEN (Дії - Реєстрація):
1. Натиснути "Signup / Login"
2. Заповнити Name: "Ivan Hurman"
3. Заповнити Email: згенерований унікальний email
4. Натиснути кнопку Signup
5. Заповнити Account Information (password, date of birth, etc.)
6. Заповнити Address Information
7. Натиснути "Create Account"
8. Побачити повідомлення "Account Created!"
9. Натиснути "Continue"

THEN (Перевірки після реєстрації):
1. URL змінився на домашню сторінку
2. У header з'явилось "Logged in as Ivan Hurman"
3. Посилання "Logout" видиме
4. Посилання "Delete Account" видиме

AND (Додаткові дії - Logout):
10. Натиснути на посилання "Logout"

THEN (Перевірки після logout):
1. URL змінився на /login
2. Текст "Logged in as" зник з header
3. Форма логіну знову видима
4. Посилання "Signup / Login" знову з'явилось у навігації

Додаткові перевірки:
- Після logout спроба доступу до захищених сторінок перенаправляє на /login
- Дані користувача більше не відображаються
